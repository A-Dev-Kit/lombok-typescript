# Architectural decisions

ADRs moved out of this file and into [docs/adr/](./adr/), one per file. Start with [docs/adr/README.md](./adr/README.md) for the index.

A single file with all 17 ADRs got unwieldy fast: linking to a specific decision from a PR meant scrolling through unrelated ones, and any update touched a huge file. Per-file ADRs follow the [adr-tools](https://github.com/npryce/adr-tools) convention used by a lot of OSS projects.

- Still open: [ADR-02 through ADR-17](./adr/README.md#index)
